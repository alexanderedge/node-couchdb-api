--- title: View categories: [api, designdocument] layout: api slug: view ---

Returns a View object for the view specified. (like Documents, they don't have to already exist in order to create a View object)

This function can also act as a setter, by passing the map and reduce functions respectively.

NOTE: You must use the save() method in order to send your changes to the database.

Arguments

name data type comments
name string The name of the view to return an object for. (or the name of the view to set)
map function Optional: The contents of the map function. You can pass a JS function, and it will be cast using toString() before storage.
reduce function Optional: The contents of the reduce function. You can pass a JS function, and it will be cast using toString() before storage. Also don't forget the "reduce function shortcuts". (_count, _sum, etc)
{% highlight javascript %} // getting a View object var view = ddoc.view("my-view"); function map(doc) { emit(doc._id, null); } function reduce(key, values, rereduce) { return values.length; } // creating a new view (w/ only a map function) ddoc.view("my-map-view", map); // creating a new view (including a reduce function) ddoc.view("my-reduce-view", map, reduce); // need to save these view views to the database ddoc.save(function (err, response) { // your new views are now available... }); {% endhighlight %}
