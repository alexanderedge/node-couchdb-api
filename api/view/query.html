Query Official Documentation


Method query
Description Queries an arbitrary view
HTTP
  • GET /db/_design/ddoc/_view/view
  • POST /db/_design/ddoc/_view/view

Arguments

Name Type Default Comments
query Optional hash Query string parameters. If only 1 of the optional args is used, this takes precedence
(See Official Documentation)
data Optional mixed If an Array, it will become the list of keys. Otherwise, (String, Hash, etc) this will become the full request body.
callback function See this page for more information 
// execute with no opts
view.query(function (err, result) {
   // your code here...
});
// specify querystring parameters (optional)
view.query({ include_docs: true }, function (err, result) {
   // your code here...
});
// specify specific keys via POST body (query must be explicitly set, or these opts will go as a querystring)
view.query({ include_docs: true }, ["id1", "id2"], function (err, result) {
   // your code here...
});
