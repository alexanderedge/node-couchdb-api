node-couchdb-api The CouchDB API Wrapper for Node.js

Code / Object Structure

Great care has been taken to leverage the powerful nature of JavaScript. This includes using prototypes for inheritance and code encapsulation, cached mixins for a smaller type of inheritance and performance as well as leveraging every object as a reference to allow for lots of useful references and discoverability.

The Client Object

Every API object has a common Client object that is the workhorse of the HTTP requests and responses. All requests are routed through here, and responses are automatically handled before being sent back to the calling objects.

var server = couchdb.srv();  // a new Client object is initialized here
server.client;               // voila!

var db = server.db("my-db"); // the client object from `server` is inherited
db.client;                   // same object as `server.client`

var doc = db.doc("my-doc");  // inherited from `db` (same as `server`)
doc.client;                  // yup, it's the same object as `db.client`

The API Objects

As mentioned before, each API object has a reference to a Client object, and inheritance is done from the Server object, down through the chain of namespaces.

As you descend into the inheritance chain, all the parent objects are also included with each instance. For example, a Database object will have a server property. A Document object will have a db property and a server property. Feel free to inspect the objects as well, there's plenty to see and use there!
